# WorkBuddy 获客 Skill v2 材料

本目录包含可安装包、可见源码、CRM 原生导入契约和设计材料。v2 保持 `crm-customer-list.v1` 不变，重点增强候选发现、产品匹配、证据质量和反向审查。

## 文件清单

| 文件 | 用途 |
| --- | --- |
| `enterprise-prospect-research_v2.skill` | 当前可安装 Skill 包。 |
| `enterprise-prospect-research/` | 与安装包一致的可见源码，包含 `SKILL.md`、`agents/`、`references/` 和 `scripts/`。 |
| `enterprise-prospect-research.skill.original.bak` | 最初版本备份，只读保留。 |
| `01-云计算获客调研设计.md` | v2 研究流程和设计边界。 |
| `02-CRM导入接口说明.md` | CRM 原生 JSON 导入规则。 |
| `schemas/crm-customer-list.v1.schema.json` | 唯一正式导入 Schema。 |
| `03-策略卡模板.md` | 产品、ICP、信号词典、通道配额和运行预算。 |
| `04-输出样例.md` | CRM 原生 JSON 虚构样例。 |
| `05-获客清单MD输出样例.md` | 面向销售的 Markdown 样例。 |

## v2 调用链

```text
产品策略卡
  → 五通道候选池（默认目标数量 × 5）
  → 主体核验、集团归属和去重
  → 公开事件证据与来源冲突处理
  → 100 分评分和负向扣分
  → 第二遍反向审查
  → CRM JSON + Markdown
  → 完整 Schema 校验 + 名单质量审计
  → CRM 预览、去重和保存
```

## 硬约束

1. JSON 顶层为 `customers[]`，`stage` 固定 `lead`，等级仅 A/B/C。
2. `fields.*` 必须是 `{ v, source, confidence, verifiedAt }`。
3. A/B 的评级依据必须有带 URL 或可定位引用的事件证据。
4. 推测写入 `businessBrief.painHypothesis` 或 `painChain`，明确“推测，未获客户确认”。
5. Skill 不生成跟进、预算、关系、已确认痛点、采购计划、联合计划或销售材料。
6. 交付前必须同时通过 `validate-crm-json.mjs` 和 `audit-prospect-quality.mjs`。

## 推荐交付

```text
crm-customer-list.v1.json   CRM 原生客户清单

```
