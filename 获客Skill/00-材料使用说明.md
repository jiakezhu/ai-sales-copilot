# WorkBuddy 获客 Skill 材料

本目录包含当前可安装包、可见源码、CRM 原生导入契约和三件套交付样例。获客 Skill 使用 `crm-customer-list.v1`，并从同一份 JSON 自动生成销售审阅 Markdown 与独立 HTML。

## 文件清单

| 文件 | 用途 |
| --- | --- |
| `enterprise-prospect-research_v3.skill` | 当前可安装 Skill 包。 |
| `enterprise-prospect-research/` | 与安装包一致的可见源码，包含指令、Schema、渲染器、校验器和样式。 |
| `enterprise-prospect-research.skill.original.bak` | 最初版本备份，只读保留。 |
| `schemas/crm-customer-list.v1.schema.json` | 唯一正式 CRM 导入 Schema。 |
| `01-云计算获客调研设计.md` | 候选发现、核验、评分和质量边界。 |
| `02-CRM导入接口说明.md` | CRM 原生 JSON 与三件套交付规则。 |
| `03-策略卡模板.md` | 产品、ICP、信号词典、通道配额和运行预算。 |
| `04-输出样例.md` | CRM 原生 JSON 虚构样例。 |
| `05-获客清单MD输出样例.md` | 由样例 JSON 自动生成的 Markdown。 |
| `sample-output/lead-list.html` | 由样例 JSON 自动生成的独立精美 HTML。 |

## 调用链

```text
产品策略卡
  → 五通道候选池
  → 主体核验、集团归属和去重
  → 公开事件证据与冲突处理
  → 100 分评分和反向审查
  → CRM JSON（唯一事实源）
  → Schema 校验 + 名单质量审计
  → 自动渲染 Markdown + HTML
  → 三文件一致性检查
  → CRM 预览、去重和保存
```

## 硬约束

1. JSON 顶层为 `customers[]`，`stage` 固定 `lead`，等级仅 A/B/C。
2. `fields.*` 必须是 `{ v, source, confidence, verifiedAt }`。
3. 每个客户必须提供 `prospectResearch` 五维评分、入选理由和反向审查。
4. A/B 评级依据必须有带 URL 或可定位引用的事件证据。
5. 推测明确标注“推测，未获客户确认”。
6. Skill 不生成跟进、预算、已确认痛点、采购计划或真实建联状态。
7. Markdown 和 HTML 必须由 JSON 自动渲染，不得单独手写。

## 固定三件套

```text
任意名称.json        CRM 原生客户清单（默认 crm-customer-list.v1.json）
lead-list.md         销售审阅、评分、逐客研判和证据追溯
lead-list.html       独立响应式网页报告，可浏览、打印和汇报
```
