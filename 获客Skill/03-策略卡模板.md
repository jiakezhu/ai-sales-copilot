# 云计算获客调研策略卡模板

这是一份给业务人员和后续生成 Skill 的 AI 阅读的策略资产。请复制后按一次获客任务填写；不要把以下可变内容写死在 Skill 提示词中。

## 1. 本次调研任务

| 项目 | 填写内容 |
| --- | --- |
| 调研任务名称 | 例如：AI 算力与云原生公开信号调研 |
| 本次销售产品/能力 | 例如：云计算基础服务、GPU 算力、容器平台、数据平台 |
| 候选企业来源 | 指定公司名单、关键词、企业信息库检索或公开市场信息 |
| 预计输出数量 | 例如：50 家 |
| 信号检索时间窗 | 默认近 180 天 |
| 强信号时间窗 | 默认近 90 天 |

## 2. V1 调研原则

- 不按企业规模、所有制、行业、客户类型或集团属性自动剔除候选企业。
- 记录这些公开属性，但不把它们当作筛选门槛。
- 只根据公开事实研究云计算场景、近期变化和证据质量。
- 后续接入内部 MCP 后，再增加客户占用排除、团队去重和目标客户范围策略。

## 3. 优先研究的云场景

按本次任务选择并排序；未选择的场景仍可作为补充发现记录。

| 优先级 | 云场景 | 观察关键词或公开线索 |
| ---: | --- | --- |
| 1 | AI 算力 / MLOps | MLOps、训练、推理、GPU、算力、数据工程、AI 功能上线 |
| 2 | 云原生 / 容器化 | DevOps、SRE、Kubernetes、云原生、平台工程、可观测性 |
| 3 | 数据平台 | 数据工程、实时计算、数据分析、数据湖、数据治理 |
| 4 | 出海部署 | 海外、跨境、多语言、全球节点、国际化 |
| 5 | 弹性流量 / CDN | 大促、开服、直播、峰值流量、带宽、低时延 |
| 6 | 安全 / 容灾 | 等保、备份、容灾、业务连续性、安全治理 |
| 7 | 云成本优化 | 降本、资源利用率、成本治理、FinOps |

## 4. 信号与评分

同一事实无论被多少个来源重复报道，只能计一次商机分；多个来源只能提高证据质量。

| 维度 | 上限 | 评分依据 |
| --- | ---: | --- |
| 业务与云负载相关性 | 40 | 公开业务、产品或交付方式与云计算场景的关联。 |
| 技术场景相关性 | 25 | 与 AI、云原生、数据、流量、出海等场景直接相关的公开证据。 |
| 购买时机信号 | 25 | 近期招聘、产品上线、扩张、采购、融资或业务变化。 |
| 证据质量 | 10 | 来源可信度、时效性、主体一致性和证据完整度。 |

### 信号分级

- 强：近 90 天出现云原生 / MLOps / GPU 相关岗位、明确云采购或迁移公告、新 AI 产品或明确海外扩张。
- 中：近 180 天融资后扩张、技术团队扩招、重要产品上线、业务量或数据规模增长。
- 弱：仅有软件、互联网或数字化业务特征。

### 分层

- A：总分至少 70，且至少一条近 90 天强信号。
- B：总分至少 50，且至少两条独立中/强信号。
- C：已有业务关联，但时机信号不足。
- manual_review：主体或公开证据不足，无法可靠排序。

## 5. 证据和输出规则

1. 每条公开事实必须有来源、抓取时间和证据摘要。
2. “可能需要某云产品”“可能存在成本压力”等只能写入需求假设，并标注置信度。
3. 未确认的真实上云现状、云账单、客户关系、预算、采购计划和痛点必须写入“待确认问题”或“未知项”。
4. CRM 导入对象只能创建为 `lead`；销售记录、客户关系、已确认痛点和方案不能自动导入。
5. 最终输出必须符合 `schemas/cloud-lead-list.v1.schema.json`。
