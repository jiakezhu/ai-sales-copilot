# 多通道候选发现

## 1. 候选池

目标候选池取 `max(目标数量 × 5, 60)`，默认不超过 300。先高召回发现，再做主体核验和评分；不要边搜索边只保留“看起来最好”的公司。

为每个候选记录：发现通道、命中查询、发现来源、标准名称候选、官网候选、初始信号和发现时间。

## 2. 五条发现通道

### ICP 圈选

使用企业库结构化条件形成市场底盘。条件来自策略卡，不自行发明硬筛选。分页获取结果，并保留总量、页数和截断情况。

### 事件驱动

围绕“产品词 + 触发事件词 + 地区/行业词”组合查询。优先原始公告、企业官网和监管/交易所信息。典型事件包括产品上线、融资后扩张、出海、迁移、合规整改和重大合作。

### 招聘驱动

搜索目标角色、技术栈和团队扩张。区分企业直招、猎头/外包转载和长期常驻岗位；同一职位的多平台转载只算一条证据。

### 采购驱动

搜索招标、采购意向、中标、框架协议、供应商招募和相关资质。区分采购方、投标方和中标方；已结束项目可证明能力或存量，但不自动代表新采购窗口。

### 相似与生态扩展

从用户样板客户、已发现高质量客户、竞品案例、上下游、集团和合作伙伴扩展。相似必须说明维度，不得只因为“同行”就复制评分或证据。

## 3. 企业 MCP 顺序

1. 企信慧眼批量发现或解析主体。
2. 天眼查核验基础画像和专项能力。
3. 企查查交叉核验关键身份字段。
4. 官网、新闻、招聘和招采原文补充事件证据。

不要为了调用更多 MCP 重复请求同一信息。关键主体字段冲突时才增加交叉来源。

## 4. 迭代规则

每轮执行：

```text
发现 → 标准化 → 去重 → 主体初核 → 通道覆盖检查 → 扩展或停止
```

最多三轮：

- 第一轮使用精确 ICP 和强信号词；
- 第二轮加入同义词、相邻角色、相邻场景和生态扩展；
- 第三轮放宽时间窗或非硬性 ICP，但不得放宽主体可靠性。

满足以下任一条件停止：

- 主体可靠候选达到目标数量的 3 倍，且五通道中至少三条有结果；
- 候选池达到上限；
- 三轮无新增有效主体；
- 工具限制或公开市场确实不足，并已记录原因。

## 5. 去重和集团处理

去重优先级：统一社会信用代码 > 标准名称 + 官网域名 > 标准名称 + 注册地。不得只按简称合并。

集团证据必须归属到实际主体。母公司战略可作为集团背景，不能直接证明子公司存在同样采购需求；子公司事件也不能复制给其他姐妹公司。

## 6. 工具失败和覆盖披露

- 查询失败可重试 2 次；改变分页或缩小查询后仍失败则切换来源。
- 工具返回截断、脱敏或无发布时间时要记录。
- 不可用来源不得以模型知识补齐。
- 报告列出每条通道的候选数、入选数、主要来源和已知盲区。
