# 信号评分与分层

## 1. 硬门槛

进入 A/B/C 前必须满足：主体可靠、企业仍可经营或状态已明确、至少一个产品相关公开事实、关键判断可追溯。否则进入 `manual_review`。

## 2. 100 分模型

| 维度 | 上限 | 评分要点 |
| --- | ---: | --- |
| 产品匹配 | 30 | 业务模式、产品形态、目标角色和产品价值是否匹配 |
| 场景证据 | 25 | 是否存在具体工作负载、技术栈或经营场景证据 |
| 购买时机 | 25 | 近期采购、招聘、上线、扩张、迁移或合规事件 |
| 证据质量 | 15 | 来源权威、主体一致、时间明确、链接可定位、交叉核验 |
| 行动可验证性 | 5 | 是否能形成具体且不预设答案的首轮确认问题 |

每个维度写分数、证据 ID 和理由。没有证据不得给“常识分”。总分截断在 0–100。

## 3. 信号强度

- **强信号**：90 天内具体采购/招标、明确迁移、与产品直接相关的关键岗位扩招、产品正式上线、明确地域或业务扩张。
- **中信号**：180 天内融资后扩张、团队扩招、重要产品发布、业务量增长、技术改造迹象。
- **弱信号**：静态行业、产品或数字化特征，没有近期变化。
- **负向信号**：项目已结束且无后续窗口、招聘为转载/外包、证据归属其他集团主体、业务收缩、产品场景明显不匹配。

同一底层事件被多个来源报道，只计一次时机分；第二来源只提高证据质量。

## 4. 默认扣分

| 情况 | 扣分 |
| --- | ---: |
| 强信号超过 90 天但仍在 180 天内 | -5 |
| 只有单一聚合来源、无原始链接 | -10 |
| 集团背景无法确认到当前主体 | -10 |
| 招聘疑似外包或长期常驻职位 | -8 |
| 采购项目已经明确结束 | -8 |
| 存在未解决的主体或事实冲突 | 移入 manual_review |

扣分不能替代事实判断；同一问题不重复扣分。

## 5. 分层

- **A**：>=75；至少 1 条 90 天内强信号；强信号有 URL/引用；反向审查通过。
- **B**：>=55；至少 2 条独立中/强信号；至少一条有 URL/引用。
- **C**：主体可靠、有产品关联，但时机不足；用于培育，不伪造强信号。
- **manual_review**：主体、归属、证据或冲突无法可靠判断。

等级不是成交概率。不要输出预算、采购周期或赢单概率。

## 6. 历史反馈校准

用户提供聚合 CRM 结果时，对比不同发现通道、场景和信号的联系率、会议率、有效商机率。只调整下一次策略卡中的关键词、通道配额和评分解释；除非用户明确批准，不改变本文件的总分结构和等级阈值。
