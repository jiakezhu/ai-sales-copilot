# 质量门槛与反向审查

## 1. 七道门槛

1. **主体门槛**：标准名称可靠；信用代码、官网或注册地至少两项可交叉定位。
2. **产品门槛**：至少一个公开事实与产品场景直接相关。
3. **证据门槛**：A/B 的评级依据必须有 URL 或可定位引用。
4. **时间门槛**：事件日期明确；无日期的内容不能作为强信号。
5. **推测门槛**：需求、痛点和方案均明确标为未确认推测。
6. **私有数据门槛**：不生成销售跟进、预算、关系、已确认痛点或采购计划。
7. **一致性门槛**：JSON、Markdown、摘要数量、名称、等级一致。

## 2. 反向审查问题

对每家拟入选客户重新检查：

- 这条证据是否属于同名企业、母公司或其他子公司？
- 多条证据是否只是同一事件的转载？
- 信号是否已经过期、项目已经结束或岗位长期重复发布？
- 是否存在与产品无关的更简单解释？
- 企业是否只是供应商/中标方，而不是采购方？
- 需求判断是否超出了公开事实？
- 如果删除最强一条证据，当前等级是否仍成立？

发现问题时修正事实、扣分、降级或转入 `manual_review`，不要只在报告中加免责声明。

## 3. 批次级覆盖检查

报告至少列出：

- 五通道候选数和最终入选数；
- 数据源成功/失败情况和截断限制；
- A/B/C/manual_review 数量；
- 有 URL 证据的客户比例；
- 90/180 天内事件覆盖率；
- 重复主体、集团证据错配和冲突数量；
- 候选池未达目标时的停止原因。

不要为了“行业多样性”机械调配名单；只检查结果是否被单一查询或来源意外垄断。

## 4. 未知项

每家客户至少记录一个真正需要销售确认的未知项。典型未知项包括现有供应商、部署方式、负载规模、决策角色、预算和采购计划。未知不等于负面，不得用模型知识补齐。

## 5. 交付失败条件

出现以下任一情况不得交付：

- JSON 校验失败；
- A 客户没有近期可定位证据；
- 存在重复企业或非法等级；
- 私有销售字段被填充；
- Markdown 与 JSON 客户数量或等级不一致；
- 工具不可用却没有披露数据盲区。
