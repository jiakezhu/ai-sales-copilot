# 腾讯 QQ 生态 UI 与客户报告重构设计

日期：2026-07-16  
状态：已完成方向确认，待书面规格复核

## 1. 目标

将云销副驾重构为同时具备两种气质的 AI 销售工作台：

- 腾讯 QQ 企鹅带来的亲和力、识别度与陪伴感；
- 腾讯 TDesign 与腾讯云控制台式的专业性、清晰度和操作效率。

本轮不是局部换色或替换图标，而是统一重做视觉层级、页面密度、核心工作流呈现和报告排版。现有业务功能与底层客户数据继续保留。

## 2. 品牌与视觉原则

### 2.1 QQ 企鹅的使用

以用户提供的腾讯 QQ 企鹅图标为唯一形象参考，不自行绘制替代企鹅。

企鹅只出现在：

- 产品品牌标识；
- 首页 AI Bot 与语音输入状态；
- AI 整理、识别、确认等关键反馈；
- 空状态、首次使用引导和轻量成功提示。

企鹅不出现在：

- 客户数据表格和指标卡内部；
- 正式客户全景报告正文；
- 高频装饰背景；
- 会削弱商务可信度的连续动画中。

### 2.2 设计语言

- 主色采用 TDesign 腾讯蓝体系，以 `#0052D9` 为主操作色，`#366EF4` 用于悬停和强调，浅蓝用于选中与 AI 状态背景。
- 页面背景为低对比浅灰，核心内容使用白色表面；减少大面积渐变、玻璃效果和厚重阴影。
- 组件采用清晰边界、小到中等圆角、紧凑但不拥挤的企业级间距。
- 强调色只服务于操作、状态和优先级；不为装饰随意增加颜色。
- 字体层级清楚，正文保持适合中文阅读的行高；数据使用等宽或更稳定的数字排版。
- 图标统一使用规范线性图标；QQ 企鹅不承担普通功能图标职责。

## 3. 信息架构与页面布局

保留一级导航：今日、客户、待办、分析。客户详情保留：作战概览、推进记录、关键关系、情报与证据。

### 3.1 全局框架

- 桌面端使用轻量深色或白色侧栏，以腾讯蓝表示当前导航；避免大面积沉重黑色。
- 顶部栏只保留全局搜索、主题/设置和主要录入动作。
- 页面内容使用统一最大宽度和 8px 间距基准，减少目前不一致的卡片大小和空白。
- PC 与移动端使用同一信息优先级，而不是简单缩放桌面布局。

### 3.2 今日首页

首页首屏围绕三个区域组织：

1. **QQ 企鹅 AI Bot**：自然语言、语音、附件和手动录入统一入口；企鹅提供人格和状态反馈。
2. **今日推进主线**：按客户价值、逾期程度和承诺日期排序，展示必须完成的下一步行动。
3. **重点客户与风险信号**：展示真正需要决策的客户变化，而不是堆叠无行动意义的指标。

指标被压缩为辅助摘要，不再占据首页主要视觉位置。

### 3.3 客户列表

- 使用清晰的 TDesign 风格表格或移动端客户卡片。
- 阶段筛选、重点等级和搜索构成单一筛选栏。
- 每行优先展示客户、当前阶段、最新推进、下一步行动和报告入口。
- 减少重复按钮；次要操作进入行尾菜单。

### 3.4 客户详情

- 客户名称、阶段、优先级和一键报告构成紧凑头部。
- 阶段轨迹使用清晰步骤条，阶段与优先级采用现代弹出选择控件。
- “推进记录”是完整销售流程时间线，明确包含沟通、任务、阶段变化、情报和材料。
- “作战概览”只展示影响当前决策的信息：机会、关系、风险、下一步和近期推进。
- QQ 企鹅不进入客户正文，只在触发 AI 整理时出现。

### 3.5 待办与分析

- 待办以完成动作效率为中心，突出逾期、今日和重点客户。
- 分析页只保留能指导行动的指标，例如阶段分布、推进停滞和客户结构。
- 图表使用腾讯蓝及状态色，不使用多彩装饰性图表。

## 4. AI Bot 交互

- 默认状态显示 QQ 企鹅、简短引导和输入区。
- 录音时使用轻微呼吸或声波动效，不让企鹅大幅跳动。
- AI 识别后生成可勾选的候选项：推进记录、客户情报和下一步行动。
- 用户确认后才写入客户档案；成功反馈由企鹅短暂出现。
- 识别失败、语音不可用或客户未匹配时提供明确恢复路径，始终允许手动填写。

## 5. 客户全景报告

### 5.1 内容原则

导出的报告只包含客户相关内容。删除：

- 产品品牌宣传；
- “由 AI 生成”或“实时汇总”等生成说明；
- 操作工具栏和返回提示；
- 无内容的字段和空白章节；
- 重复数据、无业务意义的编号和装饰页脚；
- QQ 企鹅与其他 UI 吉祥物。

### 5.2 报告结构

报告按实际有内容的章节动态生成：

1. 客户名称、报告日期、阶段与重点等级；
2. 执行摘要：当前判断、核心机会、主要风险、下一步建议；
3. 客户基本面与一线情报；
4. 决策链和关键关系；
5. 痛点、竞争态势与匹配方案；
6. 全流程推进记录；
7. 当前待办、阶段历史和攻坚计划；
8. 材料与证据索引。

无数据的章节不输出。报告内容不得补造未填写事实。

### 5.3 报告视觉

- 白色 A4 文档表面，腾讯蓝作为章节和关键结论强调色。
- 首页不做空洞的大封面，直接进入客户标题、元信息和执行摘要。
- 使用专业中文文档排版、清晰表格、克制分隔线和充足页边距。
- 关键结论采用摘要块，推进记录采用适合打印的时间线或表格。
- PDF 和 Word 保持一致的内容顺序与视觉层级。

## 6. 响应式行为

- 900px 以下切换为底部导航和单栏内容。
- AI Bot 在移动端保留输入、语音和确认，不隐藏核心能力。
- 表格在移动端转换为信息卡，避免横向压缩文字。
- 客户阶段、筛选和报告操作具备足够触摸区域。
- 报告预览在移动端单栏阅读，导出仍按 A4 版式生成。

## 7. 实现边界

- 保留当前静态前端架构、本地存储和 CloudBase 兼容逻辑。
- 保留现有客户、推进记录、任务、关系、情报和报告数据结构。
- 主要修改范围为 `index.html`、`style.css` 和 `app.js` 的渲染结构与报告模板。
- QQ 企鹅使用用户确认的图像资产；不生成或重画近似商标图形。
- 不新增与本轮视觉和报告目标无关的业务功能。

## 8. 验证标准

- 首页首屏能一眼识别 AI 输入、今日行动和重点客户。
- QQ 企鹅确实是腾讯 QQ 企鹅，并只在规定位置出现。
- 桌面端与 390px 移动端均无溢出、重叠或不可点击控件。
- AI 候选确认、语音回退、手动录入和客户写入路径保持可用。
- 客户阶段、优先级、标签页和推进时间线交互正常。
- 报告只输出有内容的客户章节，不出现产品宣传、生成说明、空字段或企鹅。
- PDF 打印和 Word 导出内容完整、层级一致、分页合理。
- JavaScript 语法检查和格式检查通过。
