# 证据与主张标准

## 证据账本

证据 ID 使用 `E001`、`E002` 递增。每条证据必须包含提供方、标题、访问时间、主体名称和可复查定位信息。网页用 `url`；MCP 可不含 URL，但必须给出 `tool_name`、`query`、`record_id` 或等价 `locator`。

`excerpt` 只保留支撑主张所需的短摘录；不得整页复制。`authority` 表示来源权威性，`directness` 表示该来源是原始记录还是转述，两者不能混为一谈。

## 主张

主张 ID 使用 `C001`、`C002` 递增。状态：

- `verified`：来源直接支持；
- `inferred`：基于事实作出的分析，必须写 `rationale`；
- `conflicted`：可信来源存在尚未消解的冲突；
- `unknown`：问题重要但没有足够证据。

置信度：`high`、`medium`、`low`、`unverified`。高置信主张至少满足其一：一个高权威直接来源；两个彼此独立的来源。媒体转载同一稿件不算两个独立来源。

## 引用完整性

所有事实记录、图谱边、事件、风险和销售假设必须引用存在的 `evidence_ids` 或 `claim_ids`。不得出现悬空 ID。证据本身不等于结论；关键结论应进入 `claims`，报告展示时同时链接证据。

## 冲突

名称、信用代码、法人、状态、持股比例、事件日期等冲突写入 `conflicts[]`，列明各说法及证据、影响和处理状态。不得选择更“顺眼”的数据静默覆盖。

## 隐私与合规

仅处理公开企业和职业信息。不得收集或输出个人手机、私人邮箱、身份证、住址、家庭关系；不得将共同任职推断为私人关系；不得绕过登录、验证码或付费权限。
