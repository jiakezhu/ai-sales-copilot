# CloudBase 接入任务文档

> 目标：把「腾讯云 · 销售获客工作台」从纯前端演示，升级为**数据云端存储 + 图片对象存储 + 手机号/微信登录**的真产品。
>
> 分工说明：**代码我（AI）已经全部改好了**，你只需要在腾讯云控制台开环境、做几项配置、把一个「环境 ID」填进代码，就能跑通。全程约 **10–15 分钟**。

---

## 〇、现在处于什么状态（重要）

代码已经改成「**双模式**」：

| 模式 | 触发条件 | 表现 |
|---|---|---|
| **演示模式** | `cloudbase-config.js` 里 `ENV_ID` 留空（**当前状态**） | 和以前完全一样：打开即用、无需登录、数据存本地。**比赛演示随时能用，不受影响。** |
| **云端模式** | 填了 `ENV_ID` | 需登录，数据存云端、图片存对象存储、多设备同步。 |

> 也就是说：**你不配置，项目照常能用；配置了，就升级成真产品。** 两者切换只靠一行 `ENV_ID`。

## 一、开通 CloudBase 环境（约 3 分钟）

1. 浏览器打开 **腾讯云 CloudBase 控制台**：<https://console.cloud.tencent.com/tcb>
2. 用你的腾讯云账号登录（你是腾讯云销售，直接用工作账号即可）。
3. 点 **「新建环境」**：
   - 计费方式：选 **「按量计费」** 或 **「包年包月-免费体验/基础版」** 均可（演示阶段免费额度够用，见文末费用说明）。
   - 环境名称：随便填，例如 `sales-crm`。
   - 地域：就近选（如「上海」「广州」）。
4. 创建完成后，进入环境，在 **「环境」→「概览」** 或页面顶部，找到 **环境 ID**（形如 `sales-crm-1a2b3c4d`）。
   👉 **把这个环境 ID 复制下来，第五步要用。**

---

## 二、开通登录方式（约 3 分钟）

进入你的环境 →左侧菜单 **「身份认证 / 登录鉴权」**（不同版本菜单名可能是「身份认证」「用户管理」）：

### A. 手机号登录（推荐，先开这个）
1. 找到 **「登录方式」**，把 **「手机号登录」** 打开（开启开关）。
2. 保存。（短信验证码由 CloudBase 内置发送，免费额度内不额外花钱。）

### B. 微信登录（可选，较繁琐，演示可先跳过）
微信登录需要额外绑定微信开放平台/公众号，配置较复杂。**建议演示阶段先只用手机号**。
如果确实要用，代码里已预留：把 `cloudbase-config.js` 的 `LOGIN.wechat` 改为 `true`，再按控制台「微信登录」指引绑定即可。

> 当前代码默认：手机号 = 开，微信 = 关。

---

## 三、创建数据库集合（约 2 分钟）

进入环境 →左侧 **「数据库」**：

1. 点 **「新建集合」**。
2. 集合名称填 **`customers`**（⚠️ 必须完全一致，代码里就是这个名字）。
3. 创建后，点这个集合 →**「权限设置」**，选择：
   **「仅创建者可读写」**（推荐，保证每个销售只能看到自己的客户数据）。
   - 若菜单是自定义安全规则，可用：
     ```json
     {
       "read": "auth.uid == doc._owner",
       "write": "auth.uid == doc._owner"
     }
     ```

> 说明：数据结构是 `{ _owner: 用户ID, list: [客户数组], updatedAt: 时间戳 }`，代码会自动创建，你**不用手动加字段**。

---

## 四、开通云存储（约 1 分钟）

进入环境 →左侧 **「云存储」**：

1. 如果是首次进入，点 **「开通」/「立即开通」**。
2. 权限设置选 **「所有用户可读，仅创建者可写」** 或 **「仅创建者可读写」** 均可（图片要在页面显示，建议“可读”宽松些）。
3. 不用手动建文件夹，代码上传时会自动放到 `sales-assets/用户ID/` 目录下。

---

## 五、把环境 ID 填进代码（约 1 分钟）⭐ 最关键

1. 打开文件：`ai-sales-copilot/cloudbase-config.js`
2. 找到第一行配置：
   ```js
   ENV_ID: "",   // 例："sales-crm-1a2b3c4d"，留空 = 本地演示模式
   ```
3. 把第一步复制的环境 ID 填进去，例如：
   ```js
   ENV_ID: "sales-crm-1a2b3c4d",
   ```
4. 保存文件。**就这一行，改完就切换到云端模式了。**

---

## 六、配置安全域名（约 1 分钟，避免登录被拦）

CloudBase 出于安全，只允许「白名单域名」调用它。你的页面部署在哪个域名，就要把那个域名加进去：

进入环境 →**「环境」→「安全配置 / 安全来源」→「WEB 安全域名」**：
- 把你的部署域名加进去，例如：
  `4ac507c9c1cd4578873d069574f5b081.tc-nanjing.share.codebuddy.woa.com`
- 本地调试的话，把 `127.0.0.1` 和 `localhost` 也加上。

> ⚠️ 不加这一步，登录/数据库调用会报「域名不在白名单」的错。

## 七、重新部署（约 1 分钟）

改完 `cloudbase-config.js` 后，需要重新部署让线上生效：
- 让我（AI）帮你重新部署到 CloudStudio 沙箱即可（一句话的事）；
- 或你自己把整个 `ai-sales-copilot/` 目录传到任意静态托管。

## 🔧 遇到问题怎么排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 打开还是不用登录 | ENV_ID 没填 / 没重新部署 | 检查第五步、第七步 |
| 登录报「域名不在白名单」 | 没配安全域名 | 做第六步 |
| 发不了验证码 | 手机号登录没开 | 做第二步 A |
| 数据存不上 / 读取失败 | 集合名不对或权限不对 | 检查第三步，集合名必须是 `customers` |
| 图片传不上 | 云存储没开通 | 做第四步 |

> 页面右上角/控制台报错时，按 F12 打开浏览器控制台看红色报错，把报错发我，我帮你定位。
